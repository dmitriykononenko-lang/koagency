# 🔗 Настройка интеграции с amoCRM

Эта инструкция поможет вам подключить калькулятор стоимости внедрения CRM к вашему аккаунту amoCRM для автоматического приема заявок.

## 📋 Что вам понадобится

- Аккаунт amoCRM с правами администратора
- Доступ к настройкам интеграций
- 10-15 минут времени

---

## 🚀 Пошаговая настройка

### Шаг 1: Создание интеграции в amoCRM

1. Войдите в ваш аккаунт amoCRM
2. Перейдите в **Настройки** (иконка шестеренки справа вверху)
3. Выберите раздел **Интеграции**
4. Нажмите **"Создать интеграцию"**

### Шаг 2: Заполнение формы интеграции

В открывшейся форме укажите:

| Поле | Значение |
|------|----------|
| **Название интеграции** | `ko:agency - Калькулятор CRM` |
| **Описание** | `Автоматический прием заявок с калькулятора стоимости внедрения` |
| **Предоставить доступ** | Выберите **"Все"** или минимум права на: <br>• Создание сделок<br>• Создание контактов<br>• Чтение списка сделок |
| **Язык** | Русский |

Нажмите **"Сохранить"**

### Шаг 3: Получение учетных данных

После создания интеграции система выдаст вам:

#### 🔑 Данные интеграции:
- **Client ID** - идентификатор интеграции
- **Client Secret** - секретный ключ
- **Redirect URI** - адрес для OAuth авторизации
- **Authorization Code** - код авторизации (действителен 20 минут)

### Шаг 4: Получение Access Token

#### Вариант A: Через Authorization Code (рекомендуется)

После создания интеграции:

1. Скопируйте **Authorization Code**
2. Выполните POST запрос для получения токенов:

```bash
POST https://ваш-аккаунт.amocrm.ru/oauth2/access_token
Content-Type: application/json

{
  "client_id": "ваш-client-id",
  "client_secret": "ваш-client-secret",
  "grant_type": "authorization_code",
  "code": "ваш-authorization-code",
  "redirect_uri": "https://ваш-redirect-uri"
}
```

Ответ содержит:
```json
{
  "token_type": "Bearer",
  "expires_in": 86400,
  "access_token": "eyJ0eXAiOiJKV1QiLCJhbGc...",
  "refresh_token": "def50200..."
}
```

#### Вариант B: Долгосрочный токен (для тестирования)

Некоторые версии amoCRM позволяют создать долгосрочный токен:

1. В настройках интеграции найдите кнопку **"Получить долгосрочный токен"**
2. Скопируйте токен (он не имеет срока истечения)

⚠️ **Внимание**: Стандартные токены живут 1 день и требуют обновления через refresh_token.

### Шаг 5: Настройка переменных окружения

В Figma Make вам нужно указать две переменные:

#### 1️⃣ AMO_DOMAIN

Это ваш поддомен amoCRM без протокола:

```
koagency.amocrm.ru
```

**Как найти:**
- Посмотрите в адресной строке браузера при входе в amoCRM
- Формат: `ваш-аккаунт.amocrm.ru`
- Если используете Kommo: `ваш-аккаунт.kommo.com`

#### 2️⃣ AMO_ACCESS_TOKEN

Это токен доступа, полученный на Шаге 4:

```
eyJ0eXAiOiJKV1QiLCJhbGciOiJSUzI1NiIsImp0aSI6IjEyMzQ1Njc4OWFiY2RlZi4uLg...
```

**Важно:**
- Токен очень длинный (500+ символов)
- Скопируйте полностью без пробелов
- Не делитесь токеном с третьими лицами

---

## 🔄 Обновление токена (для продакшена)

Access Token действует 24 часа. Для продакшн-использования настройте автообновление:

1. Сохраните **refresh_token** в безопасном месте
2. Настройте автоматическое обновление через API:

```bash
POST https://ваш-аккаунт.amocrm.ru/oauth2/access_token
Content-Type: application/json

{
  "client_id": "ваш-client-id",
  "client_secret": "ваш-client-secret",
  "grant_type": "refresh_token",
  "refresh_token": "ваш-refresh-token",
  "redirect_uri": "https://ваш-redirect-uri"
}
```

## ✅ Проверка работы

После настройки:

1. Откройте калькулятор на сайте
2. Заполните параметры расчета
3. Нажмите **"Получить КП на почту"**
4. Заполните форму с контактными данными
5. Нажмите **"Отправить заявку"**

Проверьте amoCRM:
- В разделе **"Сделки"** должна появиться новая сделка
- Название: `Заявка с калькулятора: [название пакета]`
- Бюджет: рассчитанная сумма
- Контакт с именем и телефоном клиента

## 🐛 Решение проблем

### ❌ Ошибка: "amoCRM не настроен"

**Причина:** Не указаны переменные окружения

**Решение:** 
1. Проверьте, что указали `AMO_DOMAIN` и `AMO_ACCESS_TOKEN`
2. Убедитесь, что домен указан без `https://`
3. Проверьте, что токен скопирован полностью

### ❌ Ошибка: "Unauthorized" или 401

**Причина:** Неверный или истекший токен

**Решение:**
1. Обновите токен через refresh_token
2. Проверьте права доступа интеграции
3. Убедитесь, что интеграция активна в amoCRM

### ❌ Ошибка: "Invalid grant"

**Причина:** Authorization Code истек (живет 20 минут)

**Решение:**
1. Создайте новую интеграцию
2. Сразу используйте полученный код
3. Сохраните refresh_token для дальнейшего использования

### ❌ Заявка создается в базе, но не попадает в amoCRM

**Причина:** Ошибка при отправке в API

**Решение:**
1. Проверьте логи в Supabase Functions
2. Убедитесь, что поля контакта заполнены корректно
3. Проверьте, что у интеграции есть права на создание сделок
